Station Météo Embarquée

Système de collecte et transmission de données environnementales en temps réel, basé sur microcontrôleur STM32 avec affichage local et envoi distant via protocole UART/TCP.

Contexte & Objectifs

Ce projet a été réalisé dans le cadre de ma formation en systèmes embarqués. L'objectif était de concevoir un système autonome capable de mesurer des grandeurs physiques environnementales (température, humidité, pression atmosphérique) et de les transmettre en temps réel à un système distant.

La contrainte principale était la faible consommation énergétique : le système devait pouvoir fonctionner sur batterie pendant plusieurs jours sans recharge, tout en maintenant une fréquence de mesure d'une fois par minute.

Arduino C++ — Main (setup & loop) 
// ---- SETUP ----
        void setup() {
          Serial.begin(SERIAL_BAUD);
          delay(100);
          Serial.begin(9600);
          Wire.begin();
          rtc.begin();
          SoftSerial.begin(9600);

          // Boutons et LED par défaut
          pinMode(RED_BUTTON, INPUT_PULLUP);
          pinMode(GREEN_BUTTON, INPUT_PULLUP);
          leds.setColorRGB(0, 0, 255, 0); // LED Verte — Mode Standard
          rtc.adjust(DateTime(F(__DATE__), F(__TIME__)));

          // Capteur BME280
          bme.begin();
          bool status = bme.begin();
          if (!status) {
            Serial.println(F("Erreur : capteur BME280 introuvable !"));
            indicateError(3);
          }

          loadConfigFromEEPROM();

          // Initialisation carte SD
          if (!SD.begin(SD_CS)) {
            Serial.println(F("SD init failed!"));
            while (true) {
              leds.setColorRGB(0, 255, 0, 0);   // Rouge
              delay(500);
              leds.setColorRGB(0, 255, 255, 255); // Blanc
              delay(1000);
            }
          } else {
            Serial.println(F("Carte SD prête."));
          }

          // Vérification bouton rouge au démarrage
          if (digitalRead(RED_BUTTON) == LOW) {
            modeActuel = 2;
            setLEDForMode(2);
            lastConfigActivity = millis();
          }

          applyFilenameForNow(0);

          // Interruptions matérielles
          attachInterrupt(digitalPinToInterrupt(RED_BUTTON),   onRedButtonChange,  CHANGE);
          attachInterrupt(digitalPinToInterrupt(GREEN_BUTTON), onGreenButtonChange, CHANGE);
        }

        // ---- LOOP ----
        void loop() {
          check_buttons_press(&modeActuel, &modePrecedent);

          switch (modeActuel) {
            case 1: modeStandard();      break;
            case 2: modeConfiguration(); break;
            case 3: modeMaintenance();   break;
            case 4: modeEconomique();    break;
          }
        }

Difficultés rencontrées

⚠️
Problème de synchronisation I2C

Des conflits d'adresse I2C entre le BME280 et l'écran OLED (même bus) ont nécessité une gestion fine du timing et l'ajout de résistances de pull-up adaptées (4.7kΩ).

💡
Solution apportée

Implémentation d'un arbitrage logiciel sur le bus I2C avec vérification d'état avant chaque transaction, et séquencement strict des lectures capteurs.

Ce que j'ai appris

  • Maîtrise du protocole I2C multi-esclaves sur STM32 HAL
  • Gestion des modes de veille pour l'optimisation énergétique
  • Communication AT commands avec module ESP8266
  • Débogage bas niveau avec oscilloscope (signaux I2C et UART)
  • Formatage et transmission de données JSON depuis un microcontrôleur
